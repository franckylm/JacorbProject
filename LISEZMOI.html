Ce fichier détaille la procédure d'installation et d'exécution de notre projet.

Le projet est livré avec différents scripts maven, bash et DOS qui permettent de le compiler et l'exécuter sur les trois plate-formes supportées dans les laboratoires du département, soit PC sous MS Windows (XP ou 7), PC sous Linux et Mac sous MacOS X.

Pré-requis

Le projet tel que livré ici est presque autonome. Les seuls prérequis sont que:

  • un compilateur java doit être sur le chemin d'accès ("path");
  • l'outil Maven doit être sur le chemin d'accès ("path");
  • une distribution de jacorb bien configurée doit être accessible.

Description du projet

Le but du projet est de développer un tout nouveau système policier permettant aux patrouilleurs d'obtenir le dossier d'un suspect à partir de leur voiture.

Particularités du projet

Ce projet exploite les modules maven, et sépare l'application en quatre modules:

  • idl : module contenant uniquement l'IDL;
  • server : module contenant uniquement l'application serveur et le servant;
  • client-poste : module contenant uniquement le client localisé au poste de police.
  • client-voiture : module contenant uniquement le client localisé dans les voitures de police.

Les modules server, client-poste et client-voiture dépendent du module IDL, et cette dépendance est définie au niveau de leurs POM respectifs.

Persistance des objets

Les objets sérialisés sont sauvegardés dans des fichiers d'extension .ser et localisés dans le dans le dossier persist situé à la racine du projet

Compilation de l'exemple en ciblant jacorb

  • se déplacer à l'endroit où se trouve le fichier pom.xml dans cette archive (en principe au même niveau que les présentes directives);
  • lancer mvn compile (par défaut, le compilateur IDL est exécuté, puis l'ensemble du code java est compilé).

Exécution de l'exemple avec jacorb

  • ouvrir trois coquilles (shell) selon votre plate-forme (DOS pour Windows, bash pour Unix, Linux et MacOS);
  • se déplacer (dans nos quatre coquilles) dans le répertoire qui contient les scripts pour votre plate-forme (scripts/dos pour Windows, scripts/bash pour Unix, Linux et MacOS);
  • lancer naming dans une première coquille;
  • lancer server dans une seconde coquille;
  • lancer client-poste dans une troisième coquille;
  • lancer client-voiture dans une quatrième coquille;

Intégration avec eclipse et m2e

Ce projet doit être importé dans eclipse comme projet maven. Le plugin maven pour eclipse, m2e, indique une erreur de gestion du plugin idl-maven-plugin. Ça fonctionne quand même.

Pour que ce projet compile correctement dans eclipse, vous devez:

  • ajouter le projet idl dans le "build path" des projets server, client-poste et client-voiture;
  • ajouter des répertoires sources liés ("Linked sources") à idl/target/generated-sources/idl dans ces trois projets.

Ces deux modifications s'effectuent via le "build path" eclipse.
